Extending A2UI for Agentic Knowledge Graph Visualization
In this work, I present a novel extension of the A2UI (Agent-to-UI)
framework to support Agentic Knowledge Graph visualization using
Cytoscape.js. A2UI was originally designed to bridge AI agents and
interactive user interfaces; here, it is evolved to act as a real-time reasoning-
to-visualization pipeline.
Key characteristics of this approach include:
Agent-driven graph generation
The AI agent autonomously decides which concepts become nodes and
how relationships are formed, based on its reasoning process.
Real-time, user-influenced evolution
As users interact — zooming, filtering, or asking follow-up questions —
the agent updates and restructures the graph dynamically.
Explainability by construction
The visualization directly mirrors the agent’s internal logic, making
dependencies, assumptions, and inference paths explicit.
No static schema dependency
There is no pre-modeled ontology or rigid graph structure; the graph is

What Makes These “Agentic”?
These knowledge graphs are fundamentally different from traditional static
visualizations:
1. Agent-Generated: AI agents dynamically create graphs in real-time based
on user queries, not pre-built visualizations ( look at my a2ajava project
and springa2ui to see how java returns a2ui responses )
2. Dynamic Adaptation: Graphs update and evolve as conversations
progress through dataModelUpdate messages
3. Context-Aware: Agents choose layouts, styling, and data selection tailored
to the specific request context
4. Interactive Control: User interactions (clicks, zooms) trigger userAction
events that agents respond to with updated visualizations
This “agentic” qualifier distinguishes these living, AI-controlled
